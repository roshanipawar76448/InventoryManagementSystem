# 📦 Inventory Management System

A full-stack Inventory Management System built using **React**, **Node.js**, **Express.js**, and **MongoDB Atlas**.

This project was developed as part of a technical assessment to demonstrate full-stack development skills, including REST API design, database integration, frontend state management, validation, filtering, sorting, and error handling.

---

## 🚀 Project Overview

The application enables users to efficiently manage inventory through a modern web interface. Users can create, view, update, and delete products while leveraging search, category filtering, low-stock monitoring, sorting, and validation features.

The system follows a RESTful architecture with a clear separation between frontend and backend components.

## 📊 Product Schema

Each product contains the following fields:

| Field     | Type     | Description                |
| --------- | -------- | -------------------------- |
| _id       | ObjectId | Auto-generated by MongoDB  |
| name      | String   | Product name               |
| category  | String   | Product category           |
| price     | Number   | Product price              |
| quantity  | Number   | Available stock quantity   |
| minStock  | Number   | Minimum stock threshold    |
| createdAt | Date     | Product creation timestamp |

### 📉 Low Stock Rule

A product is considered **Low Stock** when:

```text
quantity <= minStock
```

Such products are highlighted in the UI and can be fetched using a dedicated API endpoint.

## 🔗 API Endpoints

| Method | Endpoint            | Description                 |
| ------ | ------------------- | --------------------------- |
| POST   | /products           | Create a new product        |
| GET    | /products           | Retrieve all products       |
| PUT    | /products/:id       | Update a product            |
| DELETE | /products/:id       | Delete a product            |
| GET    | /products/low-stock | Retrieve low-stock products |

## ⚙️ Setup Instructions

### 1️⃣ Clone the Repository

```bash
git clone <repository-url>
cd InventoryManagementSystem
```

---

### 2️⃣ Backend Setup

Navigate to the backend folder:

```bash
cd backend
```

Install dependencies:

```bash
npm install
```

Create a `.env` file inside the backend folder:

```env
MONGO_URI=your_mongodb_connection_string
PORT=5000
```

**Note:** Create a MongoDB Atlas cluster and replace `your_mongodb_connection_string` with your MongoDB connection string.

Start the backend server:

```bash
node server.js
```

Backend runs at:

```text
http://localhost:5000
```

---

### 3️⃣ Frontend Setup

Open a new terminal and navigate to the frontend folder:

```bash
cd frontend
```

Install dependencies:

```bash
npm install
```

Start the frontend:

```bash
npm run dev
```

Frontend runs at:

```text
http://localhost:5173
```
